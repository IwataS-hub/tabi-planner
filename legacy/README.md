# 旅のしおり — 国内旅行プランナー

地名を検索してスポットを集め、地図・日程・移動時間・天気・メモを 1 画面の「しおり」にまとめる、国内旅行向けのプランニング Web アプリです。**サーバー不要・ログイン不要**で動き、データはブラウザ内（localStorage）に保存されます。GitHub Pages にそのまま公開できます。

## 主な機能

- **スポット追加**: 地名・施設名で検索（OpenStreetMap）/ 地図クリックで任意地点を追加
- **地図**: 地理院タイル（標準 / 淡色 / 写真）切替、全体表示への自動ズーム、現在地表示
- **日程管理**: Day タブで複数日に分割、ドラッグ＆ドロップで順番入れ替え
- **タイムライン**: 到着時刻・滞在時間から各スポットの目安時刻を自動計算、スポット間の移動時間・距離を表示（車 / 徒歩）
- **1 日サマリー**: 総移動時間・総距離・滞在合計・予算合計
- **情報の肉付け**: スポットごとの天気予報（7 日）＋ 服装・傘アドバイス、Wikipedia 解説と写真
- **記録**: カテゴリ分け（色分けピン）、お気に入り、設備タグ（授乳室・ベビーカー可・撮影向き 等）、メモ、予算、リンク、行った / 未踏チェック
- **持ち物リスト**: 旅ごとのチェックリスト
- **保存・共有**: 自動保存、複数の旅を切替、JSON での書き出し / 読み込み、印刷用しおりレイアウト
- **モバイル対応**: しおり ⇄ 地図のワンタップ切替

## デプロイ手順（GitHub Pages）

1. このリポジトリを GitHub に push する（`index.html` がルートにある状態）。
2. リポジトリの **Settings → Pages** を開く。
3. **Source** を `Deploy from a branch`、ブランチを `main`、フォルダを `/ (root)` に設定して保存。
4. 数分後に発行される `https://<ユーザー名>.github.io/<リポジトリ名>/` で公開されます。

ビルド不要です（React と Babel を CDN から読み込む単一 HTML）。ローカル確認は、フォルダ内で `python3 -m http.server` などを実行し `http://localhost:8000` を開くだけです。

## 技術構成

| 役割 | 使用技術 / API | 認証 | 備考 |
|---|---|---|---|
| UI | React 18 + Babel standalone（CDN） | — | ビルドレス |
| 地図 | Leaflet + 地理院タイル | 不要 | 出典表記で無料利用 |
| 地名検索 / 逆ジオコーディング | Nominatim (OpenStreetMap) | 不要 | 利用ポリシーあり |
| 経路・移動時間（車） | OSRM 公開サーバー | 不要 | デモ用・混雑時は概算へ自動フォールバック |
| 天気予報 | Open-Meteo | 不要 | 完全無料 |
| スポット解説 | Wikipedia REST API（日本語版） | 不要 | — |
| 保存 | ブラウザ localStorage | — | 端末内のみ |

すべて API キー不要・無料で動作します。

## 既知の制約

- **徒歩ルート**は直線距離からの概算（約 80m/分）です。OSRM 公開サーバーが車プロファイルのみのため、正確な徒歩経路が必要な場合は OSRM の自前ホストや有料の経路 API を検討してください。
- **鉄道・乗換**には対応していません（国内向けの無料経路 API が乏しいため）。詳細な公共交通は外部地図サービスをご利用ください。
- OSRM / Nominatim の公開サーバーには利用上限があります。本番運用や高トラフィックでは自前ホスト推奨です。
